Project Statement

Developed an iOS application leveraging Artificial Intelligence to upscale images.

Project Requirements

  • User-friendly input/output image stream on iOS.
  • Selection of multiple AI models for image upscaling.
  • API hosting image-enhancing models with server-side execution.
  • Use of AWS (Lambda Functions, S3, DynamoDB) for fast, scalable image transfer between the user and the API.
  • Automatic deletion of images after processing for data privacy.

Previous Challenges & Changes

Initially, the application design relied on an E3 AWS server to:

  • Store images
  • Run a prebuilt upscaling model
  • Send output back to the user
  • Execute cleanup tasks

However, this approach was not feasible due to high computation requirements of AI models, leading to scalability and cost issues.

The revised solution uses AWS Lambda, S3, and DynamoDB to generate secure pre-signed URLs for fast data transfer. Additionally, the application leverages the Replicate API, which specializes in server-side AI model execution, making the upscaling process faster, cheaper, and more scalable.

This approach improved security, ensured efficient communication between the user and the server, and created a scalable processing pipeline.

Tech Stack

Front-End (React Native)

  • Mobile app framework for iOS.
  • Handles image input and model selection.
  • Requests a pre-signed URL from the backend to upload images.
  • Receives the upscaled image and allows users to compare it with the original.

Back-End (Python)

  • Generates pre-signed URLs for image uploads.
  • Detects when images are uploaded to S3 and triggers the Replicate API for upscaling.
  • Stores request metadata in DynamoDB for tracking.
  • Sends the processed image back to the iOS application.

Additional Resources

  • AWS: Lambda, S3, DynamoDB for 24/7 availability, low-cost maintenance, and scalability.
  • Replicate API: Provides access to a library of AI models for image enhancement.
  • Postman: Testing API calls and cloud interactions.

Conclusion

The application was successfully built as a prototype. All core functionalities work, though the UI requires aesthetic improvements. A key challenge was managing React Native updates, which occasionally introduced breaking changes, requiring frequent syntax adjustments.
